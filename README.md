**Postgres PII Guard** — это комплексное DLP (Data Loss Prevention) решение для обнаружения, классификации и обезличивания персональных данных (PII) в базах данных PostgreSQL.

Инструмент позволяет соблюдать требования **152-ФЗ / GDPR**, безопасно передавать дампы разработчикам и проводить аудит безопасности данных.

---

## ✨ Ключевые возможности

### 🕵️ Умное сканирование (Smart Scan)
*   **Глубокий анализ:** Поиск по RegEx (Паспорта РФ, СНИЛС, ИНН, Email, Кредитки, Crypto-кошельки).
*   **Контекстный фильтр:** Отсеивание технических полей (timestamp, id, uuid) для снижения ложных срабатываний.
*   **Metadata Profiling:** Анализ названий колонок для поиска скрытых угроз (например, `user_pass`).

### 🏷️ Data Governance (Управление данными)
*   **Интерактивная разметка:** Удобный UI (Excel-like) для ручной классификации колонок.
*   **Режимы:** `AUTO` (Авто-поиск), `IGNORE` (Исключить), `FORCE_PII` (Принудительно считать угрозой).
*   **Stateful Config:** Настройки сохраняются в специальной схеме внутри самой БД.

### 🛡️ Обезличивание и Защита
*   **Masking:** Быстрое скрытие данных звездочками (`****`) через PL/pgSQL процедуры.
*   **Synthetic Data (Faker):** Замена реальных данных на реалистичные фейки (Имена, Адреса, Телефоны).
*   **Safe Dump Export:** Генерация чистого `.sql` дампа для разработчиков (система клонирует БД, обезличивает её, удаляет логи и отдает безопасный файл).

### ⚙️ Enterprise Архитектура
*   **Connection Pooling:** Высокопроизводительный пул соединений с механизмом самовосстановления (Self-Healing).
*   **Security First:** Полная защита от SQL Injection (`psycopg2.sql`), поддержка SSL (для Render/AWS).
*   **Audit Logs:** Двухуровневое логирование (App logs + Database Triggers).

## 🚀 Быстрый старт (Установка)

Для запуска требуется установленный **Docker** и **Docker Compose**.

1.  **Запуск приложения:**
    ```bash
    docker-compose up --build
    ```
    *При первом запуске база автоматически наполнится тестовыми данными.*

2.  **Доступ к интерфейсу:**
    Откройте в браузере: [http://localhost:8501](http://localhost:8501)

3.  **Остановка:**
    ```bash
    docker-compose down
    ```

После запуска откройте браузер: http://localhost:8501

## 📖 Руководство пользователя

### 1. Подключение к базе данных
Панель настроек находится в боковом меню (Sidebar).

| Тип базы | Действия | Настройка SSL |
| :--- | :--- | :--- |
| **Локальная** (Docker) | Нажмите кнопку **`📥 Загрузить из ENV`**. Поля заполнятся из `docker-compose`. | ⬜ **Выключено** (Снять галочку) |
| **Облачная** (Render/AWS) | Введите Хост, Порт, Логин и Пароль вручную. | ☑️ **Включено** (Поставить галочку) |

> 💡 **Совет:** После ввода данных нажмите **"Проверить соединение"**. Зеленая плашка подтвердит успех.

### 2. Разметка данных (Data Governance)
Прежде чем сканировать, настройте правила для колонок во вкладке **`🔭 Исследование БД`** -> **`🏷️ Управление Разметкой`**.

Вам доступна таблица-редактор. Выберите статус для каждой колонки:

| Статус | Описание | Когда использовать |
| :--- | :--- | :--- |
| **AUTO** | Система сама решает, сканировать ли колонку (пропускает `id`, `created_at`). | Для большинства полей. |
| **IGNORE** | Колонка полностью исключается из проверки. | Технические поля, логи, хеши. |
| **FORCE_PII** | Принудительная проверка указанным алгоритмом. | Для нестандартных полей (`user_login`, `token`). |

### 3. Поиск угроз (Scanning)
Перейдите во вкладку **`🚀 Сканер и Защита`**.

1.  **(Опционально)** В "Конструкторе паттернов" добавьте свои правила (например, поиск специфичного слова).
2.  Настройте **"Глубину сканирования"** в сайдбаре (слайдер). Рекомендуем `1000-5000` строк.
3.  Нажмите кнопку **`🚀 ЗАПУСТИТЬ СКАНИРОВАНИЕ`**.
4.  Изучите результаты в сводной таблице или скачайте **PDF-отчет**.

### 4. Обезличивание и Экспорт (Remediation)
В системе предусмотрено два режима защиты данных.

#### 🅰️ Режим: Безопасный Дамп (Рекомендуется)
*Создает чистую копию базы в файле `.sql`. Оригинальная база не меняется.*

1.  Прокрутите вниз до раздела **"Экспорт для разработчиков"**.
2.  Выберите метод маскирования: `Mask (****)` или `Faker (Синтетика)`.
3.  Нажмите **`🎁 СОЗДАТЬ БЕЗОПАСНЫЙ ДАМП`**.
4.  Скачайте файл. Логи аудита из него вырезаются автоматически.

#### 🅱️ Режим: Danger Zone (Изменение PROD)
*Необратимо меняет данные в подключенной базе!*

1.  Найдите красную зону **"🛡️ Меры реагирования"**.
2.  Поставьте галочку-предохранитель: `☑️ Я понимаю риски...`.
3.  Нажмите **`🧹 ОБЕЗЛИЧИТЬ ВСЕ`**.

## 🔧 Устранение неполадок (FAQ)

| Проблема | Возможная причина | Решение |
| :--- | :--- | :--- |
| **Error creating connection pool** | Неверные данные или требование SSL. | Если база облачная (Render) — включите галочку **SSL** в сайдбаре. |
| **Приложение "зависло" (Running)** | Сбой соединения WebSocket. | Обновите страницу (F5). Пул соединений сбросится автоматически. |
| **Сканер ничего не нашел** | Сработали фильтры "стоп-слов". | В "Разметке" поставьте статус **FORCE_PII** и тип `Generic` для нужной колонки. |
